# Contributing

Thanks for your interest! This project is archived and open-sourced under the
MIT License, but contributions, forks, and improvements are welcome.

## Getting started

1. Fork the repository and create a feature branch:
   ```bash
   git checkout -b feature/your-change
   ```
2. Copy any `*.env.example` files to their real names and add your own values.
3. Make your change with clear, focused commits.
4. Open a pull request describing **what** changed and **why**.

## Guidelines

- Keep pull requests small and focused.
- Match the existing code style and structure.
- Never commit secrets — real credentials belong only in git-ignored `.env` files.
- Be respectful and constructive in discussions.

By contributing, you agree that your contributions are licensed under the
repository's [MIT License](./LICENSE).
